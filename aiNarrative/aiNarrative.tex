HaLLMos is an AI-powered assessment platform that enables students to receive immediate feedback on free-response answers, including mathematical explanations and reasoning. HaLLMos activities can be embedded directly into Ximera materials, allowing students to answer open-ended, free-response questions and receive immediate AI-guided feedback. This integration extends Ximera's interactive learning environment beyond traditional answer checking to support mathematical explanation, reasoning, and reflection. Figure \ref{F:hallmos} shows feedback a student would receive for an incorrect proof. \subsection{Author Support}
AI can be instrumental in supporting new and seasoned authors through various stages of the writing process.  We present three use cases below.
\subsubsection{From a napkin sketch to a Ximera activity in five minutes} 
Authors just starting to work with the Ximera class document, can sketch their ideas on paper, upload snapshots to an LLM, and have a Ximera class document generated for them.  Figure \ref{F:lineActGPT} shows the snapshot, the prompt, the AI output, and the deployed activity. Note that the prompt requests that the LLM not only produce a Ximera class document, implying that there will be answer boxes for students to fill in, but also generate an additional similar problem.  The code generated by ChatGPT includes the author's original problem and an additional problem, all formatted for Ximera interactivity.
\begin{figure}[h]
  \centering
  \caption{ChatGPT workflow for generating a Ximera activity from a hand-sketch.}
  \includegraphics[width=.8\textwidth]{lineActivity.jpg}
  \label{F:lineActGPT}
\end{figure}
Once the code is generated, the author can upload a file with the code to their GitHub course repository and deploy the activity.
The third panel in Figure \ref{F:lineActGPT} shows the deployed activity.  In this particular case, no edits to ChatGPT's code was necessary, but changes can be easily made by the author, as desired.

\subsubsection{I can really use some Java(script)}
Some authors may wish to embed Javascript into their work for more sophisticated interactivity.  Figure \ref{F:parabola} captures the interaction with Claude by a user who is proficient in \LaTeX\ but has never written Javascript.  The third panel in the figure shows an embedded Ximera interactive produced using Javascript generated by Claude.

\begin{figure}[h]
  \centering
  \caption{Claude workflow for generating embeddable Javascript.}
  \includegraphics[width=1\textwidth]{parabolaClaude.jpg}
  \label{F:parabola}
\end{figure}

The ability to embed Javascript into Ximera is crucial for creating a direct pathway between a student's use of a gemetric manipulative and the gradebook.  Commonly embedded geometric manipulatives, such as Desmos and GeoGebra, allow for deeper learning, but do not formally record student interactions.  Having an interactive coded directly into a Ximera class document, allows direct communication between the interactive and the Ximera GODS (please fix this, Bart!).  This allows for evaluation and personalized feedback based on student's work.

\subsubsection{Supporting Authors in their use of AI}
it is important to support authors in their use of AI.  Human develpment of pedagogy and human responsiveness to student needs remains a priority.  We will support authors in their use of AI by connecting them to other Ximera authors with extensive AI experience through the Modulus Atlas program.  We will also incorporate AI training and brainstorming sessions into the annual Ximera Workshops, starting with Ximera Workship 2027.
